## 应用简介

**CRA AI Assistant** 是一款专为临床研究助理（CRA）设计的智能助手应用。它利用 GLM-4 AI 技术，帮助你从临床试验方案和受试者文档中快速提取关键信息，生成结构化的追踪表格。

### 核心功能

- **智能文档解析**：自动从 PDF 和图片中提取临床研究数据
- **结构化数据管理**：管理入选/排除标准、访视计划、受试者访视、用药记录
- **Excel 导出**：一键生成标准的临床试验追踪表格
- **本地处理**：所有数据在本地处理，保障隐私安全

## 快速开始

### 系统要求

- **操作系统**：Windows 10/11、macOS 10.15+、Linux
- **内存**：建议 4GB 以上
- **网络**：需要互联网连接（用于 AI 服务）
- **磁盘空间**：至少 200MB 可用空间

### 安装与启动

1. **安装应用**
   - 下载对应操作系统的安装包
   - 运行安装程序，按照提示完成安装

2. **启动应用**
   - Windows：双击桌面图标或从开始菜单启动
   - macOS：从应用程序文件夹启动
   - Linux：运行安装后的可执行文件

3. **配置 API Key**（首次使用）
   - 点击右上角的「设置」按钮
   - 在「API 配置」中确认 API Key（已预置默认 Key）
   - 点击「测试连接」确认配置正确
   - 点击「保存」完成配置

## 功能详解

### 1. 文件上传

#### 方案文件上传

方案文件是临床试验的核心文档，包含研究的详细设计。

**支持格式**：PDF、图片（JPG、PNG、BMP、WEBP）
**文件大小限制**：最大 50MB

**上传步骤**：
1. 将文件拖拽到「方案文件」区域，或点击区域选择文件
2. 点击「开始处理」按钮
3. AI 会自动解析文件，提取以下内容：
   - 入选标准
   - 排除标准
   - 访视计划
   - 检查项目清单

**大文件支持**：应用支持处理大型 PDF 文件。当文件超过一定大小时，会自动分批处理，并在处理过程中显示进度。

**提示**：方案文件上传后，相关工作表会自动填充数据。

#### 受试者文件上传

受试者文件包含特定受试者的访视记录、检查结果等。

**上传步骤**：
1. 切换到「受试者文件」区域
2. 拖拽或选择受试者相关文档
3. 点击「开始处理」按钮
4. AI 会识别并提取：
   - 受试者编号
   - 访视日期
   - 访视项目及结果
   - 用药记录

**大文件支持**：扫描版 PDF 会分析所有页面（而非仅首页），确保数据提取完整。

**提示**：支持批量上传多个受试者文件。

#### 文件管理

- **查看文件**：已上传的文件会显示在列表中，显示文件名、大小、上传时间
- **删除文件**：点击文件右侧的「删除」按钮可移除文件
- **清空列表**：点击「清空」按钮可一次性删除所有文件

### 2. 工作表使用

#### 入选标准工作表

**功能**：管理研究的入选标准列表

**使用方式**：
1. 从方案文件自动提取（推荐）
2. 手动添加/编辑标准
3. 导出时生成「入选标准」工作表

**字段说明**：
- **类别**：标准的分类（如年龄、诊断、实验室指标等）
- **描述**：具体的标准描述

#### 排除标准工作表

**功能**：管理研究的排除标准列表

**使用方式**：与入选标准类似

#### 访视计划工作表

**功能**：定义临床试验的访视时间表和检查项目

**使用方式**：
1. 从方案文件自动提取访视计划
2. 查看每次访视的：
   - 访视类型（如筛选、基线、随访）
   - 访视时间点（如第 -7 天、第 0 天）
   - 访视窗口期
   - 检查项目清单

**提示**：访视计划是生成受试者访视追踪的基础。

#### 受试者访视工作表

**功能**：追踪受试者的访视完成情况

**使用方式**：
1. 上传受试者文件后，AI 会自动识别受试者编号和访视记录
2. 手动补充或更正访视数据
3. 标记每次访视的状态：
   - 计划中
   - 已安排
   - 已完成
   - 缺失
   - 已取消
   - 待定

**字段说明**：
- **受试者编号**：受试者的唯一标识
- **访视类型**：对应的访视类型（从访视计划中选择）
- **计划日期**：计划执行访视的日期
- **实际日期**：实际执行访视的日期
- **状态**：访视的完成状态
- **备注**：额外的说明信息

#### 用药记录工作表

**功能**：管理受试者的用药记录

**使用方式**：
1. 从受试者文件中自动提取用药信息
2. 手动记录药物名称、剂量、用法等

**字段说明**：
- **受试者编号**：用药的受试者
- **药物名称**：药品的通用名或商品名
- **剂量**：每次用药的剂量（如 10mg）
- **频率**：用药频率（如每日两次）
- **给药途径**：如口服、静脉注射等
- **开始日期**：开始用药的日期
- **结束日期**：停止用药的日期（可选）

### 3. 数据编辑

#### 基本操作

- **添加数据**：点击「添加」按钮创建新条目
- **编辑数据**：直接点击表格单元格进行编辑
- **删除数据**：选中行后点击「删除」按钮
- **保存修改**：编辑后自动保存（如果开启了自动保存）

#### 数据验证

应用会自动验证输入数据的格式：
- 日期格式：YYYY-MM-DD
- 必填字段：标有红色星号 (*)
- 数据范围：根据字段类型自动验证

---

### 4. Excel 导出

**功能**：将所有工作表数据导出为 Excel 文件

**导出步骤**：
1. 确保所有工作表数据已填写完整
2. 点击工具栏的「导出 Excel」按钮
3. 选择保存位置和文件名
4. 等待导出完成

**导出内容**：
- 入选标准
- 排除标准
- 访视计划
- 受试者访视
- 用药记录

**文件格式**：.xlsx（Excel 2007+）

**提示**：导出的 Excel 文件包含格式化的表头、冻结的首行、自动筛选等功能，便于后续使用。

### 5. 设置管理

点击右上角的「设置」按钮打开设置对话框。

#### API 配置

| 设置项 | 说明 |
|--------|------|
| **API Key** | 智谱 AI 的 API 密钥（已预置默认 Key，可自定义） |
| **API 端点** | AI 服务的 API 地址（通常无需修改） |
| **模型名称** | 选择使用的 AI 模型（GLM-4 用于文本，GLM-4V 支持图片） |
| **测试连接** | 验证 API Key 是否有效 |

#### 处理配置

| 设置项 | 说明 |
|--------|------|
| **处理超时时间** | AI 处理的最大等待时间（30-300 秒） |
| **重试次数** | 处理失败时的自动重试次数（1-5 次） |

#### 存储配置

| 设置项 | 说明 |
|--------|------|
| **最大文件大小** | 单个文件的大小限制（1-100 MB） |

#### 界面偏好

| 设置项 | 说明 |
|--------|------|
| **主题** | 选择应用的主题（跟随系统/浅色/深色） |
| **语言** | 选择界面语言（简体中文/English） |
| **自动保存** | 是否自动保存工作表数据 |

## 常见问题

### Q1: AI 解析结果不准确怎么办？

**A**：AI 解析的准确度取决于文档的质量和格式。你可以：
1. 确保上传的文档清晰、完整
2. 手动编辑更正解析结果
3. 调整处理超时时间，给 AI 更长的处理时间

### Q2: 文件上传失败？

**A**：请检查：
1. 文件格式是否支持（PDF 或图片）
2. 文件大小是否超过限制（默认 50MB）
3. 网络连接是否正常
4. API Key 是否配置正确

### Q2.5: 支持多大的 PDF 文件？

**A**：应用支持处理大型 PDF 文件（最大 50MB），包括 500 页以上的文档。处理策略如下：

| 文件大小 | 处理方式 |
|---------|---------|
| 小文件 (<10页) | 一次性处理，速度最快 |
| 中等文件 (10-50页) | 自动分批处理，每批 5 页图片或按文本量分段 |
| 大文件 (50-500+页) | 分批处理 + 结果智能合并，每批处理后自动清理临时文件 |

**大文件处理特点**：
- 处理过程中会显示实时进度
- 每批处理完会自动释放内存，不会导致应用崩溃
- 多批次提取的结果会自动去重合并，确保数据完整

**注意**：大文件处理需要较长时间（可能 10-30 分钟），请耐心等待。

### Q3: 如何提高解析准确度？

**A**：
1. 使用高质量的扫描件或原生 PDF
2. 确保文档中的文字清晰可读
3. 对于图片，选择高分辨率的版本
4. 对于复杂文档，可以分批上传

### Q4: 导出的 Excel 文件格式不对？

**A**：确保使用 Excel 2007 或更高版本打开 .xlsx 文件。如果使用 WPS 等其他软件，可能会有兼容性问题。

### Q5: 数据会丢失吗？

**A**：应用具有以下保护机制：
1. 自动保存功能（可在设置中开启）
2. 本地存储，不上传云端
3. 建议定期导出 Excel 备份

### Q6: API Key 是什么？从哪里获取？

**A**：API Key 是调用智谱 AI 服务的密钥。
- 应用已预置默认的 API Key，可直接使用
- 如需使用自己的 API Key，请访问：https://open.bigmodel.cn/
- 注册账号后在控制台获取 API Key

### Q7: 支持哪些语言的文档？

**A**：目前主要优化了中文文档的解析。英文文档也可以处理，但准确度可能略低。
